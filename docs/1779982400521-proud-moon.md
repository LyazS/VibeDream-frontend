# 统一 exportMediaItem 和 exportTimelineItem 方案

## 背景分析

`exportMediaItem` 和 `exportTimelineItem` 存在大量重复逻辑。核心区别仅在：
- `exportMediaItem`：输入是 `UnifiedMediaItemData`，需要自行构造覆盖全长的临时 TimelineItem
- `exportTimelineItem`：输入是 `UnifiedTimelineItemData`，使用其已有的时间范围

两者对于**视频**的最终路径完全一致（构造 TimelineItem → ExportManager → MP4），仅有时间范围不同。
对于**图片**，`exportTimelineItem` 内部直接调用 `exportImageMediaItem`，已统一。
对于**音频**，`exportMediaItem` 直接返回原始文件（快速路径），`exportTimelineItem` 走 ExportManager 重编码。

## 方案：统一为 `exportSingle` + 保留原函数签名作兼容层

### 1. 新增统一 Options 接口

```ts
export type ExportSingleSource =
  | { type: 'media-item'; mediaItem: UnifiedMediaItemData }
  | { type: 'timeline-item'; timelineItem: UnifiedTimelineItemData; getMediaItem: (id: string | null) => UnifiedMediaItemData | undefined }

export interface ExportSingleOptions {
  source: ExportSingleSource
  onProgress?: (progress: number) => void
  frameRate?: number
  outputWidth?: number
  outputHeight?: number
  exportType?: ExportType  // 仅 timeline-item + video 时有效，可导出为音频
}
```

### 2. 新增核心函数 `exportSingle`

内部逻辑流程：

```
exportSingle(options)
├── 解析 source → 得到 mediaItem, resolvedTimeRange, mediaType
│   ├── type='media-item' → 直接使用 mediaItem
│   │   ┌─ image → 快速路径: canvas 绘制 → PNG Blob
│   │   ├─ audio → 快速路径: 直接返回原始文件
│   │   └─ video → timeRange = 全长(0 ~ durationN)
│   └── type='timeline-item' → getMediaItem(timelineItem.mediaItemId)
│       ┌─ image → 同上快速路径（忽略时间范围）
│       ├─ audio → timeRange = 裁剪后范围
│       └─ video → timeRange = 裁剪后范围
│
├── 对于 video/audio 需要重编码的情况:
│   └── buildCleanTimelineItem(mediaItem, resolvedTimeRange, outputSize) → ExportManager → Blob
│
└── 返回 Blob
```

提取共用辅助函数：
- `buildCleanTimelineItem(mediaItem, timeRange, outputSize)` — 构造干净的 TimelineItem
- `runExportManager(timelineItem, mediaItem, options)` — 构造 ExportProjectOptions 并执行导出

### 3. 删除旧函数，所有调用方改用 exportSingle

删除 `exportMediaItem` 和 `exportTimelineItem` 两个公开函数及其 Options 接口。
同时删除以下内部函数（逻辑合并到 `exportSingle` 中）：
- `exportVideoMediaItem`
- `exportAudioMediaItem`
- `exportImageTimelineItem`
- `exportVideoTimelineItem`
- `exportAudioTimelineItem`

保留 `exportImageMediaItem`（图片快速路径仍然独立，仅被 `exportSingle` 内部调用）。

### 4. 修改所有调用方

每个调用方从 `import { exportMediaItem, exportTimelineItem }` 改为 `import { exportSingle }`，并适配新的调用签名。

## 实现步骤

1. **新增类型** `ExportSingleSource` 和 `ExportSingleOptions`（替代 `ExportMediaItemOptions` 和 `ExportTimelineItemOptions`）
2. **提取辅助函数**：
   - `buildCleanTimelineItem(mediaItem, timeRange, outputSize, mediaType)` — 构造干净的 TimelineItem（合并 `exportVideoMediaItem` 和 `exportVideoTimelineItem` 中的重复构造代码）
   - `runExportManager(timelineItem, mediaItem, exportType, outputSize, options)` — 构造 ExportProjectOptions 并执行
3. **实现 `exportSingle`** — 按上述流程图组织逻辑
4. **修改 5 个调用方**（4 个 uploader + 1 个 composable），改用 `exportSingle` 新签名
5. **删除旧代码**：`exportMediaItem`、`exportTimelineItem`、`ExportMediaItemOptions`、`ExportTimelineItemOptions`、5 个内部函数
6. **更新 `bizyairFileUploader.ts` 的 `UploadFileExportOptions`**（目前引用了 `ExportMediaItemOptions`）
7. **验证** TypeScript 编译通过，无类型错误
8. **按测试清单逐项验证**
